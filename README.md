O Diagrama de Classes representa a vista lógica estática do sistema, fundamentada no paradigma de programação orientada a objetos (POO). A modelação destaca-se pela aplicação do princípio de herança, onde as classes Medico, Utente e Administrador derivam de uma superclasse abstrata Utilizador. Esta abstração promove a reutilização de atributos comuns (como credenciais e estado da conta) e garante uma estrita segregação de responsabilidades e permissões.
